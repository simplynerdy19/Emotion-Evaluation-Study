# Emotion Evaluation Study — MOS & EMOS Listening Test

A Streamlit web app for collecting **MOS** (naturalness) and **EMOS**
(emotion-conveyance) ratings on a 5-point Likert scale, for speech samples
generated by two TTS systems across four emotions (**angry, happy, sad,
surprise**) — 40 samples in total. Raters are **blind to which TTS system**
generated each clip; the model name is only kept in the underlying data,
never shown on screen.

Responses are stored in a **Google Sheet** (not a local file), so data
survives app restarts/redeploys if you host this on Streamlit Community
Cloud, whose local filesystem is ephemeral.

## 1. Add your audio files

Drop exactly **5 audio files** into each of these 8 folders
(`.wav`, `.mp3`, `.ogg`, `.flac`, or `.m4a` — filenames don't matter):

```
audio_samples/
├── parler_tts/
│   ├── angry/      ← 5 files
│   ├── happy/      ← 5 files
│   ├── sad/        ← 5 files
│   └── surprise/   ← 5 files
└── cosyvoice/
    ├── angry/      ← 5 files
    ├── happy/      ← 5 files
    ├── sad/        ← 5 files
    └── surprise/   ← 5 files
```

## 2. Set up the Google Sheet (one-time)

1. Go to the [Google Cloud Console](https://console.cloud.google.com/),
   create (or pick) a project.
2. Enable the **Google Sheets API** and **Google Drive API** for that
   project (APIs & Services → Enable APIs and Services).
3. Create a **Service Account** (APIs & Services → Credentials → Create
   Credentials → Service Account). Then open it → **Keys** tab → **Add
   key** → **Create new key** → JSON. This downloads a JSON file — keep it
   somewhere safe, you'll copy values out of it next.
4. Create a new Google Sheet (any name, any blank sheet is fine).
5. **Share** that Sheet with the service account's email address (it's the
   `client_email` field in the JSON file you downloaded — looks like
   `something@your-project.iam.gserviceaccount.com`), giving it **Editor**
   access.
6. Copy the Sheet's ID from its URL:
   `https://docs.google.com/spreadsheets/d/`**`THIS_PART`**`/edit`

## 3. Configure secrets

Copy the template and fill it in with your real values:

```bash
cp .streamlit/secrets.toml.example .streamlit/secrets.toml
```

Open `.streamlit/secrets.toml` and:
- Paste the Sheet ID from step 2.6 into `sheet_key`.
- Paste every field from the downloaded service-account JSON into the
  `[gcp_service_account]` section (same field names — `type`, `project_id`,
  `private_key`, `client_email`, etc.). The `private_key` value contains
  literal `\n` sequences — keep them exactly as they appear in the JSON
  file.

`secrets.toml` is already in `.gitignore` — **never commit it to GitHub.**

## 4. Install dependencies

```bash
pip install -r requirements.txt
```

## 5. Run the app

```bash
streamlit run app.py
```

This opens the study in your browser (default: http://localhost:8501). On
first load, the app will create a `responses` worksheet (tab) inside your
Google Sheet automatically, with the right header row, if it isn't there
already.

## How it works

- **Landing page**: instructions, scale descriptions, guidelines, and a
  field for the participant to type their name. It also checks the Google
  Sheet connection up front and shows a clear error (instead of crashing)
  if secrets aren't configured correctly yet.
- **Per-sample page**: plays one audio clip and asks two mandatory
  questions — MOS (naturalness) and EMOS (emotion). The model name is
  never shown — only the target emotion. The "Next" button is blocked
  with a warning until *both* questions are answered.
- **Sample order**: shuffled, but deterministically per participant name
  (so the same person always sees the same order if they resume).
- **Saving & resuming**: every answer is appended immediately as a new row
  in the Google Sheet. To resume, a participant just re-enters the exact
  same name on the landing page — the app counts how many of *their*
  samples already exist in the sheet and continues from there. Name is the
  resume key (no login/password), so ask participants to spell their name
  consistently if they plan to pause and come back.
- **Completion page**: shown once all 40 samples are rated.

## Output data

Every row in the `responses` worksheet has:

| user_name | sample_id | model | emotion | mos_score | emos_score | timestamp |
|---|---|---|---|---|---|---|

`sample_id` encodes the model, emotion, and original filename, e.g.
`parler_tts__angry__angry_01` — so even though raters never see the model
name, you can always trace a rating back to its exact source clip and
model. Open the Sheet directly any time to monitor progress, or download
it (File → Download → CSV/XLSX) for analysis in pandas/Excel/R.

## Deploying on Streamlit Community Cloud

1. Push this repo to GitHub — **make sure `.streamlit/secrets.toml` is
   excluded** (it already is, via `.gitignore`).
2. Create the app on [share.streamlit.io](https://share.streamlit.io),
   pointing at `app.py`.
3. In the app's **Settings → Secrets**, paste the exact same TOML content
   from your local `secrets.toml` (sheet_key + the full
   `[gcp_service_account]` block).
4. Deploy. Responses will now append to your Google Sheet from any
   participant, on any device, persisting across restarts/redeploys.

## Adjusting the rating scale text or study text

All study copy (instructions, guidelines, the MOS/EMOS anchor
descriptions) lives near the top of `app.py` in the `MOS_SCALE`,
`EMOS_SCALE` dictionaries and in the `landing_page()` function — edit
those directly to tweak wording.
